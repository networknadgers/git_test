How to work Github in a nutshell: The Basics




Instructions for Repositories:


On Github (The network) for forked repos not owned by you

  1. Fork the repository
  2. Clone the repository git clone "ssh of repository"
  3. Add your contributions (your code or documentation)
  4. Use git status to check the status of the modifications (should be in red)
  5. Before adding the changes to the staging area, you can check the changes you made and what is different with git diff
  6. Push your changes from "modified" area to "staged" area using git add .
    • The . at the end of the add is used to submit "all" changes. If you want specific changes do git add filename1 filename2 etc..
  7. Check status (git status) of your changes to ensure it went from 'red' to 'green' indicating it is in the "staged" area
  8. If you're happy with your changes then commit them using git commit -m "Comment about the recent edit and what you changed"
    • You can also commit without adding (git add .) every single time by committing and adding in one command: git commit -a -m "Edited changes message here"
    • If you don't add the "-m" flag with the quotations of the edit, the terminal can set it up to where VSCode automatically creates a text editor to type out your changes
      • To do this use the command on your terminal: git config --global core.editor "code --wait"
  9. Finally push your changes to the repo using git push
    • Technically for finer details, depending on the path of the repo, you would use git push origin main
  10. Create the pull request on Github and wait for it to be merged


On Github (The network) for local repos owned by you or you as an admin/collaborator

Follow steps 2-8
